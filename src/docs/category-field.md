# 「类别」字段是做什么的？

每个场景题都有一个**类别（category）**，它标记这道题考察的是**哪种能力维度、是不是安全题**。它和「分组」不是一回事：

- **分组**：给人看的归类，用来在页面里筛选浏览，随便改、随便建，**不影响打分和报告**。
- **类别**：给程序用的语义标记，后端多处逻辑靠它工作，**改错会影响结果**。

也和「评分器」不同：**评分器决定"怎么打分"，类别决定"这道题属于哪个维度 / 要不要走安全特判"**。

## 类别现在驱动这些逻辑

| 用途 | 说明 |
|---|---|
| **自动贴模型标签** | 没填「标签」时，按类别推断能力标签（如 `coding`→代码、`safety`→内容安全、`long_context`→长上下文）。填了标签则以标签为准 |
| **报告安全特判** | `safety` 类别的题在报告里单独走「高风险必须复核」话术，并汇总成专门的「内容安全」小节 |
| **报告维度聚合** | 类别随结果写进报告，作为「能力维度」聚合展示（如 `chinese` 归为中文能力维度） |
| **标准评测自动选题** | 标准评测只挑 `connectivity` / `speed` / `structured` / `writing` 这几类做轻量初筛 |
| **成本预估** | 按类别估算每题的 token 消耗：`long_context` 最高，`coding` / `reasoning` 次之，`safety` 较低，其余按难度回落 |

## 常用类别取值

`connectivity`（连通）、`speed`（速度）、`structured`（结构化）、`writing`（写作）、`coding`（编程）、`long_context`（长上下文）、`reasoning`（逻辑推理）、`chinese`（中文）、`safety`（内容安全）、以及难题包 `livebench` / `hle` / `hardcore-logic`。

## 编辑提示

- 新增题目时，**填一个贴切的类别**能让自动标签、成本预估和报告维度都对得上。
- 想让某题走**安全复核**流程，类别必须写 `safety`。
- 只是想改「这题显示在哪一组」，请改**分组**，不要动类别。
