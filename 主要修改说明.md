## 1. 报告中心 · 结论卡重构
**涉及：** `src/delivery-view.js`、`src/delivery-panel.js`、`src/app.js`、`src/styles.css`

- **准入结论卡** 不再显示"综合分"，只保留 **准入等级** 与 **成功率** 两项。
- 删除独立的"极简结论"面板，将其"结论 / 原因 / 下一步"内容做成 **极简结论卡**，替换原"最近请求"卡片的位置。
- 报告结论卡上方新增一条 **请求成功 / 失败状态行**（成功为绿、失败为红）。当最近一次请求失败时，相关内容统一显示"最近一次请求失败"。
- 新增 `derivePlainConclusion()`——用**本地规则**生成"大白话结论"，不额外调用模型。

## 2. 报告生成后自动在浏览器打开
**涉及：** `server/report-files.mjs`、`server/task-manager.mjs`、`server.mjs`

- 新增 `openReportInBrowser()` / `isOpenReportEnabled()`：测试生成 HTML 报告后，自动用系统默认浏览器打开。
- Windows 下用 `explorer.exe` 的**完整路径**（`%SystemRoot%\explorer.exe`）启动，规避 `ENOENT`；进程错误事件静默处理、`unref()` 不阻塞。
- 覆盖快检、单渠道准入、稳定性、批量、场景等各类报告产出路径。

## 3. 模型官方价目「参考」表
**涉及：** `server/model-prices.mjs`（新增）、`tests/model-prices.test.mjs`（新增）

- 录入 OpenAI GPT-5.x 与 Anthropic Claude（Fable / Opus / Sonnet / Haiku 等）官方单价。
- 提供 `lookupModelPrice()`（容忍 `. - _` 分隔与大小写）、`listModelPrices()`。
- **明确定位为"参考"**：中转 / relay 渠道实际价常与官方标价不同，本表**不参与成本估算的静默兜底**，渠道真实成本仍以各渠道自填单价为准。

## 4. 全量提示词按 PTCF 框架重写
**涉及：** `server/scenarios/*.mjs`（basic / chinese / coding / long-context）、`server/judge-orchestration.mjs`、`server/ai-report-analysis.mjs`、`src/prompt-presets.js`、`server/token-auditor.mjs` 等

- 依据 `write-prompt` skill 的 **Persona / Task / Context / Format** 结构，重写软件中的各处提示词，使指令更清晰、可控。
- 严格保留所有被测试断言依赖的字符串与评分关键词，行为不变。

## 5. 界面折叠与精简
**涉及：** `index.html`、`src/styles.css`

- 场景测试页："查看场景测试摘要和报告位置"改为折叠条；"批量并发测试"改为折叠条；"查看说明"类入口置顶。
- 稳定性测试页：去掉"报告摘要与文件路径"标题与"复制摘要和路径"按钮，仅保留折叠条「查看测试摘要和报告位置」（同时移除 `app.js` 中对应的失效监听）。

## 6. 被测目标选择器升级（此改动来自dev
**涉及：** `src/target-picker.js`（新增）、`src/batch-target-picker.js`（新增）、`src/scenario-case-picker.js`（新增）、`src/app.js`

- **级联选择器**（渠道 → 模型）：单目标运行页统一改为"先选渠道、再选该渠道下模型"的级联下拉。
- **批量两维度选择器**：渠道体检（一渠道·多模型）/ 渠道选优（一模型·多渠道）两种维度，勾选 + chips 展示。
- **场景多选改造**：场景测试页"选择测试场景"由原生 `<select multiple>` 列表框改为与"勾选要体检的模型"**同款**的勾选列表 + 全选/清空 + chips 样式；真值仍写回隐藏的 `<select multiple>`，既有提交读取与"测试模板自动勾选"逻辑不变。
