## 11. Claude 分词器指纹核验（准入评测）
**涉及：** `server/tokenizer-probes.mjs`（新）、`scripts/claude-token-baseline.mjs`（新）、`scripts/claude-token-baseline.json`（新，生成产物）、`server/tokenizer-fingerprint-audit.mjs`（新）、`server/test-runner.mjs`、`server/reporting.mjs`、`src/admission-view.js`、`tests/tokenizer-fingerprint-audit.test.mjs`（新）、`package.json`、`.env.evaluator.example`

- **原理**：对一组固定探针文本，先在可信源上建立"输入 token 数基线"；准入时把被测渠道返回的 `usage.prompt_tokens` 与基线做线性拟合 `reported ≈ slope·base + intercept`。同一代分词器 slope≈1、R²≈1（模板固定开销只改 intercept）；换了分词器（挂羊头 / 换代）则 slope 偏离或 R² 掉。判据以 **|slope−1| 为主、R² 为辅**；只判"代 / 家族"，不区分同代具体型号。
- **触发**：仅当被测模型声称 Claude 家族（`inferModelFamily`）且本地有该代基线时；探针用 `max_tokens=1`、不带 temperature、不写日志，产出成本极低。被测模型不可能"自己数 token"，指纹只取响应里的 usage 字段。
- **展示**：准入结果模块新增「分词器指纹」卡（与流式结构 / 慢请求参考同行、同格式）；准入报告"3. 关键指标"加一行，并新增"## 10. 分词器指纹核验（Claude 身份）"章节。
- **分词器代**：脚本自动按探针向量分代；实测分出 2 代 —— `[claude-opus-4-8]`（含 4.7 / Fable5）与 `[claude-opus-4-6 + claude-sonnet-4-6]`。
- **诚实边界**：基线可能建在可信中转（非官方分词），判定只依赖"线性关系"而非绝对相等；整套核验最多与基线来源一样可信。已在基线 JSON、报告章节、卡片小字三处声明。
- **交叉验证留档**：用**两个互相独立的可信账号**建基线 / 核验，在 opus-4.7 代与 opus-4.6 代上均得 **slope=1.000、R²≈1**（仅模板开销不同，被 intercept 完整吸收）→ 互证 committed 基线确为真 Claude 分词。反向对照：DeepSeek 官方 slope≈0.58；某掺假账号返回自相矛盾乱数（长文本反而更少 token）——均被判出。方法对"真货 / 别家族 / 伪造 / 错代"四类区分均有效。
